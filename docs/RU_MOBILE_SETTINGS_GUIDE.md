# Настройки мобильного приложения VPN: что делает каждый пункт и как лучше для России

Этот файл можно переслать пользователям как практическую памятку.
Текст написан простым языком: что включает настройка, когда нужна, когда не нужна и насколько часто реально помогает.

Проценты ниже не "математика из лаборатории", а практическая оценка по реальным кейсам пользователей (в каких случаях опция действительно улучшает работу).

## Как читать этот документ

- "Полезность" = в каком проценте проблемных случаев настройка реально помогает.
- "Нужно" = обычно стоит оставить включенным для РФ.
- "Не нужно" = обычно не трогать, включать только под конкретную проблему.
- Если после изменений стало хуже: верните спорные пункты к дефолту и проверьте снова.

## 1) General (Общие)

### Language (Язык)
- Что делает: меняет язык интерфейса приложения.
- Когда нужно: только для удобства.
- Полезность: 0% для сети (100% для удобства).
- Статус: `Нужно` по личному предпочтению.

### Theme mode (Тема)
- Что делает: светлая/темная/системная тема.
- Когда нужно: только визуально.
- Полезность: 0% для сети.
- Статус: `Не влияет`.

### Analytics
- Что делает: отправка обезличенной статистики (если включено).
- Когда нужно: если хотите помогать сбором статистики.
- Полезность: 0% для разблокировки.
- Статус: `Необязательно`.

### Auto IP check
- Что делает: после подключения проверяет внешний IP, чтобы понять, поднялся ли туннель.
- Когда нужно: почти всегда полезно для контроля.
- Полезность: 65%.
- Статус: `Нужно`.

### Dynamic notification (Android)
- Что делает: показывает живой статус VPN в уведомлении.
- Когда нужно: для контроля подключения.
- Полезность: 20% (диагностика, не лечение).
- Статус: `Желательно`.

### Haptic feedback (Android)
- Что делает: виброотклик интерфейса.
- Когда нужно: кому так удобнее.
- Полезность: 0% для сети.
- Статус: `Не влияет`.

### Disable battery optimization prompt (Android)
- Что делает: убирает напоминание об оптимизации батареи.
- Важный момент: если само приложение не добавлено в исключения батареи, Android может выгружать VPN в фоне.
- Полезность: 55% для проблем "само отваливается".
- Статус: `Нужно` добавить приложение в исключения; подсказку можно выключить после этого.

### Memory limit
- Что делает: ограничивает память core-процесса.
- Когда нужно: только если устройство очень слабое и есть перегрев/дефицит памяти.
- Риск: слишком маленький лимит = обрывы, нестабильность.
- Полезность: 25%.
- Статус: `Обычно не нужно трогать`.

### Debug mode
- Что делает: расширенный режим отладки.
- Когда нужно: только во время диагностики.
- Полезность: 35% (только для поиска причины).
- Статус: `Включать временно`.

### Log level
- Что делает: детализация логов (`error`, `warn`, `info`, `debug`).
- Когда нужно: `info` для обычной жизни, `debug` только на время разбора проблемы.
- Полезность: 40% (диагностика).
- Статус: `Нужно`, но без крайностей.

### Connection test URL
- Что делает: адрес для проверки доступности сети/прокси.
- Практика: лучше ставить стабильный и "нейтральный" сайт, который часто не блокируется.
- Полезность: 45%.
- Статус: `Желательно`.

### URL test interval
- Что делает: как часто выполняется проверка из пункта выше.
- Оптимум: не слишком часто (лишняя нагрузка), не слишком редко (медленная реакция).
- Полезность: 35%.
- Статус: `Желательно`.

### Clash API port
- Что делает: локальный API-порт для внешних панелей/автоматизации.
- Когда нужно: только если вы точно используете Clash API.
- Полезность: 10%.
- Статус: `Обычно не нужно`.

### Use Xray core when possible
- Что делает: где возможно, использует Xray-движок вместо sing-box.
- Когда нужно: при проблемах совместимости с конкретным протоколом/сервером.
- Полезность: 30%.
- Статус: `Оставить по дефолту`, менять точечно.

## 2) Routing (Маршрутизация)

### Region
- Что делает: выбирает региональные правила (geo-логика, direct/proxy приоритеты, DNS-поведение).
- Для РФ: это один из ключевых пунктов.
- Полезность: 90%.
- Статус: `Критично нужно` поставить `Russia`.

### Balancer strategy
- Что делает: стратегия выбора узла при нескольких серверах.
- Когда нужно: если у вас группа узлов и авто-выбор.
- Полезность: 50%.
- Статус: `Нужно`, если серверов несколько.

### Block ads
- Что делает: блокирует рекламные/трекерные домены правилами.
- Риск: иногда ломает вход в приложения и часть контента.
- Полезность: 45%.
- Статус: `Опционально`; если что-то "странно не грузится", сначала выключить для проверки.

### Bypass LAN
- Что делает: локальная сеть (роутер, NAS, принтеры, локальные IP) идет мимо VPN.
- Полезность: 75%.
- Статус: `Нужно`.

### Resolve destination
- Что делает: заранее резолвит домен для точного матчирования роут-правил.
- Плюс: иногда исправляет неверный выбор маршрута.
- Минус: иногда добавляет лишние DNS-запросы и побочки.
- Полезность: 35%.
- Статус: `Не трогать без причины`.

### IPv6 mode
- Что делает: политика обработки IPv6.
- Для РФ-практики: если есть нестабильность/частичная недоступность, чаще помогает режим с приоритетом IPv4.
- Полезность: 60%.
- Статус: `Нужно настроить аккуратно` (обычно `prefer_ipv4` в связке с DNS-стратегией).

### Per-app proxy (Android)
- Что делает: VPN не для всех приложений, а выборочно.
- Режимы:
  - `off`: одинаково для всех приложений.
  - `include`: только выбранные приложения через VPN.
  - `exclude`: все через VPN, кроме выбранных.
- Полезность: 70% в кейсах "часть приложений должна идти напрямую".
- Статус: `Нужно` только при смешанном сценарии; иначе `off`.

## 3) DNS

### Remote DNS address
- Что делает: DNS для трафика внутри VPN-логики.
- Для РФ: часто рабочий вариант `tcp://8.8.8.8`, если ваш провайдер/сеть не ломает этот путь.
- Полезность: 85%.
- Статус: `Критично важно`.

### Remote DNS domain strategy
- Что делает: как выбирать A/AAAA.
- Практика: `prefer_ipv4` обычно стабильнее для проблемных сетей.
- Полезность: 70%.
- Статус: `Нужно`.

### Enable Fake DNS
- Что делает: fake-ip механизм для прозрачного перехвата.
- Плюс: полезен в некоторых туннельных сценариях.
- Минус: у части банков/гос/корп-приложений вызывает странные ошибки.
- Полезность: 30% (узкоспециализированно).
- Статус: `Для РФ обычно не нужно`, лучше `OFF`.

### Direct DNS address
- Что делает: DNS для direct-трафика (который не идет через прокси).
- Для РФ: максимально безопасно `local` (системный/локальный резолвер), чтобы не ломать локальный контекст.
- Полезность: 95%.
- Статус: `Обязательно` для стабильности российских сервисов.

### Direct DNS domain strategy
- Что делает: стратегия доменного резолва для direct-контра.
- Практика: `prefer_ipv4`.
- Полезность: 65%.
- Статус: `Нужно`.

## 4) Inbound (Режим туннеля)

### Service mode
- Что делает: базовый тип работы приложения.
- На мобильном обычно:
  - `tun` = полноценный VPN.
  - `proxy` = локальный прокси-режим.
- Для обычного использования VPN на телефоне: нужен `tun`.
- Полезность: 95%.
- Статус: `Обязательно`.

### Strict route
- Что делает: делает маршрутизацию жесткой и "без компромиссов".
- Риск: может ломать локальный доступ и часть российских сервисов при неудачных правилах.
- Полезность: 25% (редкие кейсы).
- Статус: `Для РФ обычно OFF`.

### TUN implementation
- Что делает: реализация виртуального сетевого стека.
- Общая практика:
  - `system` сначала.
  - если нестабильно, пробовать `mixed`.
  - `gvisor` только если есть конкретная причина.
- Полезность: 60%.
- Статус: `Нужно подобрать один раз`.

### Mixed port / Direct port
- Что делает: локальные порты для прокси-компонентов.
- Для обычного пользователя: не трогать.
- Полезность: 15%.
- Статус: `Обычно не нужно`.

### Allow connection from LAN
- Что делает: разрешает другим устройствам в вашей сети подключаться к вашему локальному прокси.
- Риск: безопасность, если включено без необходимости.
- Полезность: 20%.
- Статус: `Выключено`, если не нужен LAN-доступ.

## 5) TLS Tricks (обход DPI)

### Enable TLS fragment
- Что делает: дробит TLS-пакеты для обхода DPI/фильтрации.
- Плюс: помогает при "режут handshake".
- Минус: может ухудшить скорость и стабильность.
- Полезность: 55%.
- Статус: `Включать только при симптомах DPI`.

### Fragment packets mode
- Что делает: профиль фрагментации.
- Практика: начинать с самых "мягких" режимов, не ставить агрессивный сразу.
- Полезность: 45%.
- Статус: `Только при необходимости`.

### Fragment size
- Что делает: размер фрагментов.
- Полезность: 40%.
- Статус: `Тонкая настройка`.

### Fragment sleep
- Что делает: пауза между отправкой фрагментов.
- Полезность: 35%.
- Статус: `Тонкая настройка`.

### Mixed SNI case
- Что делает: меняет регистр в SNI для обхода примитивных фильтров.
- Полезность: 25%.
- Статус: `Опционально`.

### Enable TLS padding / TLS padding size
- Что делает: добавляет "шум" в TLS-пакеты.
- Плюс: иногда помогает против сигнатур DPI.
- Минус: лишний трафик и задержка.
- Полезность: 30%.
- Статус: `Только если без этого не работает`.

## 6) WARP

### Enable WARP
- Что делает: добавляет Cloudflare WARP как дополнительный транспорт/слой.
- Плюс: иногда помогает обойти часть ограничений.
- Минус: добавляет сложность и может конфликтовать с основной схемой.
- Полезность: 35%.
- Статус: `Для базового РФ-сценария обычно не нужно`.

### Generate WARP config
- Что делает: генерирует конфиг WARP.
- Полезность: 15% (только если используете WARP).
- Статус: `Опционально`.

### WARP detour mode
- Что делает: порядок прохождения трафика между WARP и прокси.
- Полезность: 20%.
- Статус: `Только для продвинутой схемы`.

### License key / Clean IP / Port / Noise*
- Что делает: тонкая конфигурация WARP-канала.
- Полезность: 10-25%.
- Статус: `Обычно не нужно`.

## Что точно нужно / что обычно не нужно (для России)

## Нужно (базовый стабильный набор)
- `Service mode = tun`
- `Region = Russia`
- `Bypass LAN = ON`
- `Direct DNS address = local`
- `Remote DNS address = tcp://8.8.8.8` (или DNS вашего сервера)
- `Remote DNS strategy = prefer_ipv4`
- `Direct DNS strategy = prefer_ipv4`
- `Strict route = OFF`
- `Auto IP check = ON`
- `TUN implementation = system` (если плохо -> `mixed`)

## Обычно не нужно (без конкретной проблемы)
- `Fake DNS`
- `TLS fragment/padding` на постоянной основе
- `WARP`
- `Resolve destination`
- `Clash API port` и ручная правка портов inbound
- `Debug mode` (постоянно)

---

## Полезность настроек в процентах (короткий рейтинг)

### Очень полезные (80-95%)
- `Direct DNS = local` -> 95%
- `Service mode = tun` -> 95%
- `Region = Russia` -> 90%
- `Remote DNS выбран корректно` -> 85%

### Средние (50-79%)
- `Bypass LAN` -> 75%
- `DNS strategy prefer_ipv4` -> 65-70%
- `TUN implementation (подбор system/mixed)` -> 60%
- `Per-app proxy` при смешанном сценарии -> 70%

### Нишевые (20-49%)
- `TLS tricks` -> 25-55% (только при DPI)
- `Block ads` -> 45% (может и помочь, и сломать)
- `Resolve destination` -> 35%
- `WARP` -> 35%

### Почти не влияет на доступ к сайтам (0-19%)
- `Language`, `Theme`, `Haptic` -> 0%
- `Clash API port` / ручные порты без нужды -> 10-15%

## Лучшая настройка "против блокировок и DPI" (рабочий практический пресет)

Этот профиль стоит ставить первым. Он дает стабильность для РФ и не перегружает конфиг "магией":

1. `Service mode: tun`
2. `Region: Russia`
3. `Direct DNS: local`
4. `Remote DNS: tcp://8.8.8.8` (или DNS сервера, если он стабильнее)
5. `DNS strategy (remote/direct): prefer_ipv4`
6. `Strict route: OFF`
7. `Bypass LAN: ON`
8. `Fake DNS: OFF`
9. `WARP: OFF`
10. `TLS fragment + padding: OFF` (включать только если есть явные признаки DPI)

Если блокировка именно DPI и базовый пресет не помог:

1. Включить `TLS fragment`.
2. Проверить 1-2 щадящих режима фрагментации (без агрессивных параметров).
3. При необходимости включить минимальный `TLS padding`.
4. Проверять после каждого шага, а не включать все сразу.

Почему так: сначала выстраиваем правильный DNS и маршрутизацию (это решает большую часть проблем), и только потом добавляем DPI-обход, если реально нужен.
